# Sky Defender: Joe's Story — Remaster Specification (pix3)

> Спецификация разработки ремастера Flash-игры **Sky Defender: Joe's Story** (2010–2011,
> GritsenkoWeilard) на движке **pix3**. Основана на оригинальном GDD v1.12
> ([gdd/gdd-sky-defender-v1.12-ru.md](gdd/gdd-sky-defender-v1.12-ru.md)), игровых конфигах
> ([original-data/](../design/original-data/)) и декомпиляции последней сборки
> `cannon_game_v10.18` (AS3). Исходники Flash (.fla) не сохранились; вся механика
> восстановлена из GDD, XML-конфигов и декомпилированного кода.

---

## 1. Что это за игра

**Жанр:** side-view tower defense + аркадный шутер. Игрок **сам** целится и стреляет из
«главного калибра» — многоствольной зенитной пушки на башне-замке в левой части экрана.
Волны воздушных и наземных врагов наступают справа. Между волнами — магазин: этажи замка,
автотурели, апгрейды и новое оружие.

**Сеттинг:** мир Гайя — парящие острова «небожителей» над погибшей Землёй. Варвары с
выжженной поверхности поднимаются в небо на ржавых дирижаблях, чтобы отвоевать «украденное
небо». Джо — пилот парящей башни, последняя линия обороны провинции. Тон — юмор и пародия
(Джо ворчит про «однобортные сюртуки», реклама в магазине — злая сатира, смерть Джо —
спрайт T-800).

**Столпы (из GDD):**
1. **Пиротехника** — взрывы как в кино о WWII: горящие дирижабли падают факелами с хвостами дыма.
2. **Разрушение объектов** — составные юниты разбираются по частям, обломки падают на мост.
3. **Юмор** — потешные враги-пародии, шутки в диалогах и магазине.

## 3. Поток экранов

```
Preloader → Intro (диалог Короля и Джо, textures/intro/) → Main Menu
Main Menu → Campaign (карта мира) | Survival | Options | Achievements | Awards | Credits
```

## 4. Ядро геймплея

### 4.1 Замок (левая колонна экрана)

4 этажа, покупаются в магазине. HP по этажам: **700 / 1000 / 1300 / 1600 (+100 бонус)**.
Спрайты: `textures/castle/floor_gate.png, floor_1..4.png` + повреждённые варианты `floor_*d.png`.

| Этаж | Слоты |
|---|---|
| Floor 1 | — (желоб подачи ядер) |
| Floor 2 | 1 наземная турель; сюда переезжает главная пушка |
| Floor 3 | 1 наземная турель |
| Floor 4 | 1 воздушная (AA) пушка |

Турели: `TR1` — dmg 10, период 30 тиков, дальность 500; `TR2` — dmg 10, период 80, дальность 500
(из `conf.xml <TRS>`). Флаг (+HP, анимация `textures/interactive/flag/flg0000-0022.png`),
«жилки»-прочность моста/стен: HP 200/400/700/1000 (`<ZHILKI>`).

Дополнительно из магазина: **Umbrella** (щит, режет урон при HP<75%), **Fire support**
(авиаудар каждые 30 с при HP<50%), **Crazy mineman** (авто-мины на мосту, respawn 5 с).

### 4.2 Главный калибр

Пушка на подвижной раме, наведение мышью. Сектор обстрела **±65°** (тюнить до 60°).
Мёртвые зоны — осознанная механика: мотивируют покупать турели.

| # | Оружие | Аналог | Урон L1/L2 | Магазин | Боезапас | Особенность |
|---|---|---|---|---|---|---|
| 1 | Gun | пистолет | 50/70 | 3 | 70 | простая дуга |
| 2 | Shotgun | дробовик | 25/30 → 40/80 (Fire) | 5 | 14 | веер картечи; Fire-вариант — детонация |
| 3 | Mini-gun | «вертушка» | 50/70 | 2×100 | 100 | цепочка ядер по одной траектории |
| 4 | Rifle | снайперка | 100/150 → 200/400 | 4 | 50/2 | луч-указка через весь экран, выстрел по прямой, долгая перезарядка |

(урон/боезапас — из `conf.xml <DMG>/<AMMO>`; перекладываются в баланс-конфиг ремастера)

**Баллистика** (из декомпила v10.18, 30 fps): снаряд получает
`dirx = cos(angle)·V`, `diry = sin(angle)·V`; каждый кадр `diry += G; x += dirx; y += diry·0.025`.
Gun/Rifle стреляют «жёстче» (V≈320 по вертикальной компоненте), Shotgun/Minigun — V≈25 с
выраженной дугой. В ремастере — то же через фиксированный шаг ECS, константы в конфиге.

**Процедурная анимация пушки:** откат ствола на 2-3 px при выстреле, секвенция «нагнетания»
(`textures/maingun/gun/00000-00005.png`), вспышка `fx.png` рисуется **и под, и над** стволом
(дубляж кадра), гильзы у Shotgun. Отдача: Джо отъезжает на 3 px влево.

**Джо-аватар** (`textures/npc/joe/`, спека в [gdd/joe-animation-notes-ru.md](gdd/joe-animation-notes-ru.md)):
- **Idle** — «дыхание»: 3 px вверх / 6 вниз / 6 вверх, цикл + моргание (`eye.png` поверх).
- **Shoot** — отдача + наложение `*F.png` (засвет от выстрела).
- **Dead** — замена на `dead.png` (T-800).
- **Laugh** — при убийстве босса/крупного корабля: дыхание ×2-3 скорости + смеющийся спрайт.

### 4.3 Противники

Реестр 48 типов — [original-data/units.txt](original-data/units.txt); классификация — GDD §Противники
и схемы в [references/classification/](references/classification/).

**Воздух** (`textures/enemy/air/`): Lucky/Slevin (бомберы), Little Avalon ×4, Avalon ×4,
Lavalon ×8 (истребители/торпедоносцы разных конфигураций), NZ_1-5, SUC_1-4 (typical_bloon —
рядовые аэростаты), Unik/Urik (уникальные юниты со стропами), support-юниты
(Fatty, Fish, Splash, S, S_Nut), Transporter (тянет сегмент моста).

**Земля** (`textures/enemy/ground/`, идут по мосту): atabus, attaban, baka, baron, bb, bus,
dream, dreamer, fatima, garbag, medic, rracer, siege, warchild + пушки gun1/gun2.

**Боссы** (`textures/enemy/bosses/`): **Baby, Grafz, Rud, Snake, Xenon** — каждый с
повреждённым (`*_d.png`) и «вспышечным» (`*_white.png` — кадр урона) вариантами.
Уничтожение — по ключевым точкам (слотам), последовательно.

**Классы поведения:** камикадзе, бомбардировщик, истребитель, торпедоносец, таран,
пушка/танк, бонус-юнит, мини-босс (Lancaster, Largo), босс.

**Составные юниты (Multi Unit)** — фишка игры: аэростат = носитель + стропы + гондола, у
каждой части свой хитбокс и сценарий разрушения:
- попал в носитель → полная детонация;
- попал в стропы → стропы рвутся, шар улетает вверх, гондола падает (может взорваться на мосту и задеть наземных!);
- попал в гондолу → гондола взрывается, шар со стропами улетает.

Наземные юниты — монолитные (Simple Unit).

**Оружие врагов:** лёгкая пушка (дуга, отдача), тяжёлая (прямая наводка), мортира (крутая
дуга, 3 типа ядер); снаряды: мина (на стропах у возд.), чугунное/каменное/огненное ядро
(огненное — с хвостом), торпеда (свободное падение → включение двигателя → по прямой,
эффект littlebg + след). Спрайты: `textures/enemy/weapons/`.

**Мост:** 3 транспортёра приносят 3 сегмента (`enemy/ground/bridge1/00000-00011.png` —
анимация смятия), после последнего — звук рога, начинается наземная атака. Мост
неразрушаем; сегменты — «отсчёт 1-2-3» перед штурмом.

### 4.4 Волны и миссии

Формат оригинала (`original-data/mobs.xml`, семантика из декомпила):
```xml
<Lvl n="2">
  <Mob>
    <t>5</t>      <!-- время спавна, секунд от старта волны (в коде ×30 кадров) -->
    <id>39</id>   <!-- тип юнита по units.txt (39 = Unik_2_1) -->
    <y>240</y>    <!-- высота спавна (коорд. 640×480) -->
    <a>320</a>    <!-- x-позиция, с которой юнит открывает огонь / встаёт на позицию -->
    <tip>1</tip>  <!-- вариант поведения/комплектации -->
    <dop>0</dop>  <!-- доп. параметр (высота атаки, груз и т.п.) -->
    <com>zxc</com><!-- метка-комментарий волны -->
  </Mob>
</Lvl>
```
`presets.txt` — готовые паттерны построений («клин S-wing», «крест», «галка»).
В ремастере — тот же формат как JSON/YAML `waves.json` на миссию; спавнер — ECS-система.

**Экономика миссий:** награды за миссии — [gdd/mission-golds-ru.txt](gdd/mission-golds-ru.txt)
(за скорость, за «ни один слиток не украден», за выживание NPC и т.п.); очки за юниты —
`conf.xml <Mob><score>`; время волн `<TIMELEVEL>` (80/120/120 c).

### 4.5 Магазин

скриншот — [references/screens/shop.jpg](references/screens/shop.jpg)):
этажи 1-4, флаг, турели 1-2, AA-пушка, Umbrella, Fire support, Mineman, 4 оружия + апгрейды

### 4.6 Спецэффекты (визитная карточка игры)

Все эффекты — пререндеренные секвенции (`textures/sfx/`), не частицы:

| Эффект | Ассеты | Механика |
|---|---|---|
| Типичный взрыв | `sfx/boom1/` (22 кадра) + `blowglow.png` | 3 копии секвенции под случайными углами (A/B/C: поверх юнита, за ним) + BlowGlow растёт и тает |
| Падение с горением | `sfx/fire/`, `sfx/fire2/` (23 кадра), `fireb/` | остов падает «кометой», 2 очага огня (57%/100% альфы), ориентация очагов = траектория |
| Удар о мост | damaged_shapes/*.png | развал на ≤5 кусков по пологим дугам, каждый с мини-огнём, куски падают спереди и позади моста |
| Множественная детонация | боссы | цепочка взрывов нос→корма + глобальный BlowGlow размером с босса; зеппелин падает МЕДЛЕННО, величественно |
| Тление | `*_d.png` + fire1/fire2 | обгоревший остов, очаги с фейд-ином, наклон очагов против движения |
| **Взрывная волна** | `sfx/explosion_wave.png` | маска 321×321, blend **Overlay** (fallback normal 37%); растёт ×2 с затуханием. **Геймплейная**: расталкивает соседние юниты от эпицентра; юнит, толкнутый в башню/мост — взрывается. Волна = скрытое оружие игрока |
| Ореол детонации | blowglow | подсветка соседних объектов при взрыве (иллюзия света, без шейдеров) |

В ремастере: секвенции → `AnimatedSprite2D`; Overlay-блендинг → blending-режим
материала спрайта (three.js `AdditiveBlending`/custom blend); расталкивание — ECS.

### 4.7 Небо и параллакс

`textures/backgrounds/level_0/`: `level_0.jpg` (задник) + `cloud1-8.png`.
Из `original-data/sky.xml`: ширина мира **2560 px** (при экране 640), 4 слоя облаков со
скоростями **0.7 (передний) / 0.57 / 0.27 / 0.17**, у переднего слоя альфа ~0.7, облака
перекрывают весь контент кроме UI. Расстановка облаков — в sky.xml.

## 7. Маппинг на движок pix3

| Механика оригинала | Реализация в pix3 |
|---|---|
| Сцены-экраны (menu/map/game/shop) | отдельные `.pix3scene` + `scene.changeScene(..., { transition: 'fade' })` |
| Слои: небо → задник → сцена → передние облака → HUD | порядок в дереве 2D (draw order = tree order); HUD — `CanvasLayer2D` |
| Спрайты юнитов | `Sprite2D`; повреждённые/white-кадры — смена texture |
| Секвенции (взрывы, огонь, флаг, винты) | `AnimatedSprite2D` |
| Параллакс-облака | `Camera2D` + скрипт `ParallaxLayers` (множители скоростей из sky.xml) |
| Составные юниты (шар/стропы/гондола) | prefab-сцена: Group2D с дочерними Sprite2D-частями, каждая со своим хитбоксом-компонентом |
| Игровой цикл, баллистика, волны, коллизии | **ECS** (`ECSService`, fixed-step) — снаряды/юниты как entities; спавнер читает waves-конфиг |
| Наведение пушки | скрипт: `this.input` (позиция мыши) → угол рамы, clamp ±65° |
| Взрывная волна-расталкивание | ECS-система импульсов (без Rapier — простая 2D-математика, как в оригинале) |
| HP-бары, боссбар | `Bar2D` |
| Кнопки меню/магазина | `Button2D` (normal/hover/pressed = `x.png`/`x_o.png`) |
| Тряска камеры при попадании в замок | `Camera2D.shake` + juice |
| Диалоги/брифинги | UI-сцена + скрипт-тайпрайтер; портреты — Sprite2D |
| Катсцена-интро | Cutscene Director / keyframe animation (core:AnimationPlayer) |
| Сейвы (прогресс кампании, деньги, ачивменты) | localStorage-сервис в user-скрипте |
| Баланс | `design/original-data/*` → `src/assets/data/balance.json` + `waves/*.json` |

Правила: логика — `Script`-компоненты (`user:*`) в `scripts/`; сначала смотреть
`core:*`-поведения; ассет-пути `res://src/assets/...`; текстуры 2D прогоняются через
`configure2DTexture` движком автоматически.

### Замечание по разрешению
Оригинал — 640×480, ассеты низкого разрешения. Проект настроен на 1920×1080. Стратегия v1:
глобальный масштаб игрового поля ×2.25 (мир строится в оригинальных координатах внутри
Group2D-«стейджа» со scale), фильтрация linear. Позже: точечный AI-апскейл ключевых
спрайтов (замок, пушка, боссы) через встроенный Asset Generator с оригиналами как
style-reference.
