# Pix3 Editor — инструкция по редизайну под новый макет
Для агента, работающего в кодовой базе редактора (кастомный фреймворк на Lit-компонентах). Эталон — макет `Pix3 Editor.html` (+ `styles.css`, `chrome.jsx`, `viewport.jsx`, `scene-tree.jsx`, `inspector.jsx`, `asset-browser.jsx` в этом проекте). Цель: привести цвета, вкладки, тулбары, иконки и общий вид к новому, более чистому дизайну — без изменения логики.

---

## 0. Принципы нового дизайна
- **Плоско и тихо.** Никаких градиентов на хроме, никаких выпуклых кнопок. Панели различаются на 1 ступень фона + 1px линия.
- **Один акцент.** Мятный (`--accent`) — только для: выделения, активного инструмента/вкладки, primary-кнопки (Play). Всё остальное — нейтральные серые.
- **Разделение линиями, не тенями.** Тени — только у плавающих элементов (тулбары во вьюпорте, поповеры, палитра команд).
- **Плотность:** базовый шрифт 13px, строки списков 22–26px, высота заголовка панели 32px, топбар 40px, статусбар 24px.
- **Радиусы маленькие:** 4/6/8px. Пилюли (999px) — только для тегов/аватаров/пресенса.

## 1. Токены — единый источник правды
Завести один глобальный CSS-файл с токенами (скопировать из `styles.css` макета) и подключать его в document root. **Кастомные CSS-свойства проникают в shadow DOM Lit-компонентов** — поэтому токены объявляем на `:root` документа, а внутри компонентов используем только `var(--...)`.

```css
:root {
  /* Поверхности (холодные нейтралы) */
  --bg-0: oklch(0.16 0.008 250); /* фон приложения, вьюпорт-хром */
  --bg-1: oklch(0.19 0.008 250); /* панели */
  --bg-2: oklch(0.22 0.008 250); /* raised: инпуты, hover строк */
  --bg-3: oklch(0.26 0.008 250); /* hover raised, активный сегмент */
  --bg-4: oklch(0.31 0.008 250);
  /* Линии */
  --line-1: oklch(0.28 0.008 250);      /* все разделители панелей */
  --line-2: oklch(0.34 0.008 250);      /* борта поповеров, hover-борта */
  --line-strong: oklch(0.42 0.008 250);
  /* Текст */
  --fg-0: oklch(0.97 0.005 250);  /* основной */
  --fg-1: oklch(0.82 0.006 250);  /* вторичный, иконки по умолчанию */
  --fg-2: oklch(0.62 0.006 250);  /* третичный, лейблы полей */
  --fg-3: oklch(0.48 0.006 250);  /* muted, счётчики, хинты */
  /* Акцент */
  --accent: oklch(0.78 0.15 160);
  --accent-soft: oklch(0.78 0.15 160 / 0.16); /* фон выделения */
  --accent-line: oklch(0.78 0.15 160 / 0.42); /* борт активных элементов */
  --accent-fg: oklch(0.18 0.04 160);          /* текст НА акцентном фоне */
  --danger: oklch(0.7 0.18 25);
  --warn: oklch(0.82 0.15 80);
  /* Presence (курсоры/аватары коллабораторов) */
  --presence-1: oklch(0.78 0.15 160); --presence-2: oklch(0.74 0.16 30);
  --presence-3: oklch(0.78 0.15 280); --presence-4: oklch(0.82 0.15 80);
  --presence-5: oklch(0.74 0.13 220);
  /* Тип */
  --font-ui: "Segoe UI", "Roboto", system-ui, sans-serif;
  --font-mono: "Cascadia Code", "Roboto Mono", "Consolas", ui-monospace, monospace;
  /* Геометрия */
  --row-h: 26px; --pad-x: 10px;
  --radius-1: 4px; --radius-2: 6px; --radius-3: 10px;
  /* Тени — только для плавающих элементов */
  --shadow-pop: 0 1px 0 oklch(1 0 0 / 0.04) inset, 0 8px 24px oklch(0 0 0 / 0.4);
  --shadow-float: 0 1px 0 oklch(1 0 0 / 0.05) inset, 0 12px 40px oklch(0 0 0 / 0.5), 0 0 0 1px var(--line-1);
}
```

**Правила миграции цветов:**
- Найти и удалить ВСЕ hex/rgb-литералы в стилях компонентов; заменить маппингом на токены. Ничего «похожего на глаз» не оставлять.
- Старые синие/голубые акценты → `--accent` (мятный). Красный destructive → `--danger`.
- Если в Lit-компонентах есть свои темы/токены — сделать их алиасами новых (`--old-panel-bg: var(--bg-1)`), затем постепенно выпилить.
- Скроллбары: глобально, thumb `--line-2`, hover `--line-strong`, трек прозрачный, ширина 10px, radius 10px (см. `styles.css`).

## 2. Особенности Lit (важно)
- Токены наследуются через shadow DOM автоматически; **стили — нет**. Общие паттерны (кнопка-иконка, вкладка, инпут) оформить как разделяемые `css`-литералы (`export const buttonStyles = css\`...\``) и включать в `static styles` компонентов, либо как маленькие переиспользуемые Lit-компоненты (`<px-icon-button>`, `<px-tab-bar>`, `<px-field>`).
- В каждом компоненте задать `:host { font: 13px/1.4 var(--font-ui); color: var(--fg-0); }` где уместно; `box-sizing: border-box` для всего.
- `user-select: none` на всём хроме (панели, тулбары, вкладки); выделяем только текстовый контент/инпуты.
- Фокус: `:focus-visible { outline: 2px solid var(--accent); outline-offset: 1px; }` — единый focus-ring, убрать нативные outline/borders.
- Не использовать `!important`; специфичность держать низкой, чтобы токены-переопределения работали.

## 3. Каркас приложения
Сверху вниз:
1. **TopBar** — 40px, `--bg-1`, нижний борт `--line-1`.
2. **Рабочая зона** — 3 колонки: дерево сцены слева (260px, правый борт), центр (вьюпорт + нижняя док-панель ассетов 240px, верхний борт), инспектор справа (304px, левый борт).
3. **StatusBar** — 24px, `--bg-1`, верхний борт, моношрифт 10.5px, цвет `--fg-2`.

Все разделители — строго `1px solid var(--line-1)`. Никаких двойных бортов, теней между панелями, и никаких зазоров между панелями (единая сетка).

### TopBar (см. `chrome.jsx → TopBar`)
- Слева: логотип + «Pix3» (13.5px, 700) → хлебные крошки `Org / scene` (12.5px, разделитель «/» цветом `--fg-3`, текущий пункт `--fg-0`, остальные `--fg-1`) → пилюля ветки git (ghost-пилюля, mono 10.5px).
- Центр: командная строка-кнопка (⌘K): max 380px, h28, `--bg-2`, борт `--line-1`, radius 7, плейсхолдер `--fg-2`, справа `<kbd>`-чипы.
- Справа: стек аватаров (перекрытие −6px, кольцо `0 0 0 1.5px var(--bg-1)`) · вертикальный делитель · **Play** (единственная залитая акцентом кнопка: `--accent` фон, `--accent-fg` текст, h28, radius 7, 600) · Share (h28, `--bg-2`, борт `--line-1`).

## 4. Вкладки — три вида, не смешивать
1. **Вкладки панелей** (заголовки Scene/Layers/Outline, Assets/Prefabs/Console): в шапке панели h32; сами вкладки — текст 11.5px/500, UPPERCASE не нужен для файловых, для панельных — uppercase c letter-spacing 0.02; неактивная `--fg-2`, активная `--fg-0` + **подчёркивание 2px `--accent`** (inset слева/справа 6px, radius 2, прижато к нижнему борту шапки). Без фоновых плашек, без бортов у вкладок.
2. **Вкладки документов** (открытые сцены над вьюпортом): ряд h32 на `--bg-1`; активная вкладка — «карточка» h24, `--bg-0` (фон вьюпорта), борт `--line-1`, radius 5, внутри: точка-индикатор 6px `--accent` (несохранённые изменения) + имя файла 11.5px `--fg-0` + крестик `--fg-3`. Неактивные — просто текст `--fg-2` без фона. Кнопка «+ New» — призрачная, `--fg-2`. Справа в этом же ряду — ghost-пилюли (Snap, Grid).
3. **Сегмент-переключатели** (2D/3D и т.п.): контейнер `--bg-0`, radius 6, padding 1; активный сегмент — заливка `--accent`, текст `--accent-fg`, 11px/600, radius 5; неактивный — прозрачный, `--fg-1`.

## 5. Иконки
- Единый набор: SVG 16×16 (у крупных 18), stroke `currentColor`, stroke-width 1.5, round caps/joins, без заливок (см. `icons.jsx`). Выпилить растровые/разнородные иконки.
- Цвет по умолчанию `--fg-1`; в выделенной строке/активном состоянии — `--accent`; muted — `--fg-3`. Никогда не жёсткий цвет в самой SVG.
- **IconButton** — базовый атом (сделать `<px-icon-button>`): 26×26, radius 6; default прозрачная, иконка `--fg-1`; hover `--bg-2` + `--fg-0`; active/toggled: фон `--accent-soft`, борт `1px solid --accent-line`, иконка `--accent`; danger — `--danger`. Transition 120ms на background/color. Обязателен `title` с названием и хоткеем («Move  W»).

## 6. Тулбары вьюпорта — плавающие, не встроенные
Убрать встроенные полосы инструментов; инструменты — плавающие «островки» поверх канвы:
- **Островок = карточка:** `--bg-1`, борт `1px solid --line-1`, radius 8, padding 3–4, `box-shadow: var(--shadow-pop)`, отступ от края канвы 12px.
- **Слева-сверху, вертикально:** инструменты (Select/Move/Rotate/Scale, делитель `1px --line-1` c margin 2px 4px, затем Pan/Frame/Pen/Comment) из IconButton'ов, gap 1. Активный инструмент — ровно один.
- **Сверху по центру:** сегмент 2D/3D + тумблеры видимости слоёв (h22, 10.5px/600; вкл: `--bg-3` фон + `--fg-0`; выкл: прозрачный + `--fg-3`; борт `--line-1`).
- **Справа-сверху:** зум-группа: −, значение (mono 11px, min-width 42, по центру), +, делитель, fit-to-view.
- **Слева-снизу:** нав-гизмо (круг 78px, оси X/Y/Z — красный/мятный/голубой бейджи 16px).
- **Справа-снизу:** координаты/fps — mono 10.5px `--fg-2`, значения `--fg-0`, та же карточка-островок.
- Фон канвы: `--viewport-grid`-паттерн — база `oklch(0.13 0.008 250)`, сетка белым 2.5% (16px) и 5% (80px).

## 7. Панели: дерево, ассеты, инспектор
**Шапка панели** (общий компонент `<px-panel-header>`): h32, `--bg-1`, нижний борт; вкладки слева (см. §4.1) или заголовок (11px/600 uppercase `--fg-1` + счётчик mono `--fg-3`); справа — группа IconButton'ов gap 1.

**Строки списков/деревьев** (дерево сцены h24, файлы/папки h22):
- default: прозрачный фон, текст `--fg-0` (файлы `--fg-1`), иконка `--fg-2`;
- hover: `--bg-2`;
- selected: фон `--accent-soft`, текст и иконка `--accent`, **левый борт 2px `--accent`** (у невыделенных — 2px transparent, чтобы не прыгало);
- muted/скрытые ноды: `--fg-3`; отступ вложенности ~10–12px/уровень; presence-точка 6px цветом коллаборатора с кольцом `--bg-1`.

**Ассет-браузер:** слева дерево папок 170px c правым бортом; сверху мини-тулбар h28 (поиск, вид сетка/список); тамбы на шахматке (`checker`), radius 4, борт `--line-1`, выделенный — `1.5px solid --accent`, подпись 10.5px (`--accent`/600 у выбранного). Drop-зона — `1px dashed --line-2`, radius 6, текст `--fg-3`.

**Инспектор:**
- Секции (`Section`): заголовок 10.5px/600 uppercase ls 0.06 `--fg-1`, шеврон-раскрывашка `--fg-3` (rotate 90° открыто, transition 150ms), нижний борт `--line-1`; контент padding 2px 12px 12px.
- Поле (`Field`): grid `70px 1fr`, gap 8; лейбл 11px `--fg-2`.
- Инпуты: h22, `--bg-2`, борт `--line-1`, radius 4, 11.5px; числовые — mono, `tnum`, серые префиксы X/Y/W/H внутри. Чекбокс 13px, radius 3, checked = заливка `--accent` + галочка `--accent-fg`.

## 8. Оверлеи
- **Command palette (⌘K):** ширина 560, `--bg-1`, борт `--line-2`, radius 10, `--shadow-float`; подложка `oklch(0 0 0/0.55)` + `backdrop-filter: blur(4px)`; выбранный пункт — `--accent-soft` + левый борт 2px `--accent`; группы — 10px uppercase `--fg-3`; хоткеи — `<kbd>`-чипы (18px, mono 10.5, `--bg-2`, борт `--line-1`, radius 4).
- Поповеры/меню/тултипы — те же правила карточки: `--bg-1`/`--bg-2`, борт `--line-2`, radius 8–10, `--shadow-pop|float`.

## 9. Порядок работ (по PR)
1. **Токены:** глобальный файл, алиасы старых переменных → новые; глобальные скроллбары, focus-ring, шрифты. Визуально уже «перекрасит» большую часть.
2. **Атомы:** `<px-icon-button>`, `<px-tab-bar>` (все 3 вида вкладок), `<px-pill>`, `<px-kbd>`, инпуты. Заменить по всему коду.
3. **TopBar + StatusBar.**
4. **Панели:** panel-header + строки дерева/списков (дерево сцены, ассеты, инспектор).
5. **Вьюпорт:** вкладки документов, вынос тулбаров в плавающие островки, сетка фона, зум/гизмо/статус-островки.
6. **Оверлеи:** палитра, меню, тултипы.
7. **Зачистка:** grep по hex-цветам, `box-shadow` вне островков/оверлеев, старым радиусам >10px, шрифтам ≠ токенам.

## 10. Definition of done / чек-лист
- [ ] Ни одного hex/rgb-литерала в компонентах — только `var(--...)`.
- [ ] Акцентный цвет встречается только в: выделении, активных вкладках/инструментах, Play, focus-ring, presence-you.
- [ ] Все панельные разделители 1px `--line-1`; тени только на плавающих элементах.
- [ ] Все иконки — единый stroke-набор через `currentColor`.
- [ ] Вкладки соответствуют одному из 3 паттернов §4; нигде нет «кнопочных» вкладок с заливкой.
- [ ] Строки: hover `--bg-2`, selected `--accent-soft` + левый борт 2px.
- [ ] Высоты: топбар 40 / шапки панелей 32 / строки 22–26 / инпуты 22 / статусбар 24.
- [ ] Числа — mono + tabular-nums; UPPERCASE-лейблы ≤11px c letter-spacing.
- [ ] `:focus-visible` виден на каждом интерактивном элементе; hit-таргеты ≥ 26px.
- [ ] Тёмная тема единственная; контраст текста `--fg-2` на `--bg-1` проверен (≥ 4.5:1 для основного текста).
