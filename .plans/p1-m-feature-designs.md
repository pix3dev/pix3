## 1. Decisions (each fork, with mechanism)

### D1 — World-space simulation: inverse-world compensation on `renderRoot` (not scene-root reparenting)

**Chosen:** keep `renderRoot` parented to the node; when `simulationSpace === 'world'`, set `renderRoot.matrixAutoUpdate = false` and each frame write `renderRoot.matrix = this.matrixWorld.clone().invert()` so `renderRoot`'s effective world transform is identity. Particle state is then stored/composed **directly in world coordinates**.

**Rejected:** reparenting the instanced mesh to the scene root — breaks node lifecycle (removal, visibility inheritance via `isObjectVisibleInHierarchy` in SceneRunner 1026, play-mode clone teardown, editor selection raycasts) for no math benefit.

**Why not "transform world→local per instance":** node scale/rotation would distort particle size and billboard orientation; the inverse-matrix trick neutralizes the whole ancestor chain in one matrix.

**Mechanism details:**
- New private `syncWorldCompensation()`: `this.updateWorldMatrix(true, false); this.tempMatInv.copy(this.matrixWorld).invert(); this.renderRoot.matrix.copy(this.tempMatInv);`. Guard non-invertible (node scale 0): if `det≈0`, skip (keep last).
- Called at top of `tick()` (so spawn math uses a fresh matrix) **and** again in the render-time sync method (D4) to latch late ancestor motion (scripts/AnimationPlayer that tick after this node, editor gizmo drags).
- **Spawn:** `assignSpawnPosition(target)` unchanged (emitter-shape local offset), then in `activateParticle`: if world mode, `particle.position.applyMatrix4(this.matrixWorld)` (offsets correctly inherit node scale) and `particle.velocity.applyQuaternion(this.getWorldQuaternion(tempQuatWorld))` (direction only; speed magnitude not scaled).
- **Update:** unchanged — gravity, integration, color/alpha all operate in "sim space" (world when world mode). Gravity semantics become world-vector in world mode, which is what users expect.
- **Billboard:** in `applyBillboard`, when world mode, `tempQuatNode.copy(cameraQuaternion)` directly (renderRoot world = identity) instead of the inverse-node-quat product at lines 241–242.
- **Space switching at runtime:** new method `setSimulationSpace(next: 'local'|'world')` — sets field, calls `restart()`, resets `renderRoot.matrix.identity()` + `matrixAutoUpdate = (next === 'local')`. Schema setter (line 808–813) changes to call it. Loader/constructor path just sets the field before first tick, no restart needed.
- **Prewarm:** constructor prewarm runs before the node has a parent → `matrixWorld` is local-only. Acceptable (prewarm at authored origin); note in docs. Skip sub-emission during prewarm (see D3).

**Regression flag (required by brief):** every scene saved since `simulationSpace` shipped persists the field (Saver writes it unconditionally, line 637). Any external scene authored with `simulationSpace: world` silently behaved as `local`; after this change its particles stop following the emitter (now-correct behavior). No migration — the field finally does what its label says — but this MUST be called out in the PR description and `docs/node-types-reference.md`. In-repo footprint: zero scenes affected (verified above).

### D2 — Trails: per-particle ring-buffer ribbons in one shared `Mesh`, camera-facing, RGBA vertex colors

**Chosen:** one `THREE.Mesh` (`trailMesh`) under `renderRoot` (so D1 compensation applies for free), `BufferGeometry` with pre-allocated buffers for `maxParticles × trailSegments` samples, rebuilt (positions/colors) every rendered frame. Unused ribbon area collapses to degenerate triangles (all verts at head position) — static index buffer, no per-frame index churn.

**Rejected:** per-particle `Line2`/mesh objects (allocation churn, draw calls); GPU trails (no compute infra, the whole node is CPU-sim); `THREE.Line` (1px lines, no width on most platforms).

**Material:** separate `MeshBasicMaterial({ transparent: true, depthWrite: false, blending: AdditiveBlending, side: DoubleSide, vertexColors: true })` with a **4-component `color` attribute** — three.js auto-defines `USE_COLOR_ALPHA` when `geometry.attributes.color.itemSize === 4`, so per-vertex alpha needs **no `onBeforeCompile`** (unlike the instanced material's `instanceAlpha` hack at 460–485). Untextured in v1.

**Data (allocated only while `trailEnabled`):**
- `trailData: Float32Array(maxParticles * trailSegments * 3)` — ring buffer of sim-space sample positions.
- `ParticleState` additions: `trailHead: number; trailLen: number; trailTimer: number` (reset in `activateParticle`).
- Geometry: `position` Float32 (`maxParticles * trailSegments * 2 * 3`), `color` Float32 itemSize 4, prebuilt `index` (`maxParticles * (trailSegments-1) * 6`). All `DynamicDrawUsage`.

**Sampling math (in `updateParticles`, per active particle, after integration):**
```
interval = trailLifetime / (trailSegments - 1)
p.trailTimer += dt
while p.trailTimer >= interval:
    p.trailTimer -= interval
    p.trailHead = (p.trailHead + 1) % trailSegments
    write p.position into trailData[particleIdx][p.trailHead]
    p.trailLen = min(p.trailLen + 1, trailSegments)
```

**Ribbon build (render-time, needs camera — see D4), per particle:**
```
// sample 0 = live head (current p.position), samples 1..len walk ring backwards
for i in 0..len-1:
    P = sample(i)
    D = normalize(sample(i-1) - sample(i+1))        // central diff; clamped at ends
    V = normalize(camPosSim - P)                     // camPosSim: camera world pos,
                                                     // transformed by matrixWorld⁻¹ when local mode
    side = normalize(cross(D, V)) * (trailWidth/2) * (1 - t)   // linear taper, t = i/(segments-1)
    vertA = P + side ; vertB = P - side
    rgba = (particleCurrentColor, particleAlpha * (1 - trailFade * t))
for i in len..segments-1: write head position to both verts (degenerate)
```
Inactive particles: collapse their whole strip to `(0,0,0)` with alpha 0 once on deactivation (dirty-write, not per-frame).

**Lifecycle:** `restart()` zeroes `trailLen` for all particles + collapses geometry; `setMaxParticles()` and new `setTrailSegments()` re-allocate all trail buffers and rebuild `trailMesh` geometry; disabling `trailEnabled` removes/disposes `trailMesh` and frees `trailData` (setter method `setTrailEnabled(v)`).

**Memory guard:** `trailSegments` clamped 2–64; buffers exist only when enabled. Worst legal case (10 000 × 64) ≈ 1.28 M verts ≈ 15 MB buffers + per-frame CPU rebuild — document "keep maxParticles moderate with trails" in the schema `description`; no hard product clamp beyond the 64.

Trails work in both sim spaces; in `local` on a moving emitter the whole ribbon rides with the node (technically consistent, visually "wrong"). Schema description for `trailEnabled` says: "Best with Simulation Space = world."

### D3 — Sub-emitters: death-triggered burst into a **referenced** `Particles3D` node, deferred to post-loop

**Chosen:** reference another `Particles3D` by node **id** (`subEmitterId`, schema type `'node'`, `ui.nodeTypes: ['Particles3D']`), copying `VirtualCamera3D.resolveTarget` exactly (scene lookup + root-walk fallback, cached per resolved id). On particle death, the parent records the death position/velocity (world space) into a per-frame scratch list; **after** the update loop it calls `sub.emitBurstAt(...)` for each death. No collision triggers, no inline spawning mid-iteration.

**Why a separate node, not inline config:** the burst particles need their own material/texture/color/lifetime — that's a whole second emitter's worth of knobs; nesting them flat would double the schema. Godot does exactly this (`sub_emitter` NodePath). The referenced node is typically authored with `emissionRate: 0`.

**Why deferred:** firing during `updateParticles` iteration would mutate the target (possibly `this`, possibly a node already ticked this frame) mid-loop. Deferring makes self-reference and any tick-order safe; burst particles become visible on the target's next `updateParticles` (≤1 frame latency, invisible in practice).

**New public API on `Particles3D`:**
```ts
/** Spawn `count` particles around a world-space point. Safe to call from scripts. */
emitBurstAt(worldPosition: Vector3, count: number, inheritedWorldVelocity?: Vector3): void
```
Implementation: convert `worldPosition` (and velocity direction) into the target's sim space (`world` mode: as-is; `local` mode: `applyMatrix4(matrixWorld⁻¹)` / inverse world quaternion for velocity); then for up to `count` inactive particles call `activateParticle(p, originSimSpace, velocityBias)` — `activateParticle` gets two optional params: after `assignSpawnPosition` add `originSimSpace` (shape offset becomes offset *around* the death point) and skip the node-matrixWorld spawn transform of D1 (origin is already sim-space); then `p.velocity.addScaledVector(velocityBias, 1)` where the caller pre-scaled by `subEmitterInheritVelocity`.

**Parent-side hook (end of `updateParticles`, replacing the bare `particle.active = false` at line 395–398):** when a particle dies and `subEmitterId !== ''`, push `[x,y,z,vx,vy,vz]` (world space: particle state as-is in world mode; `position.applyMatrix4(this.matrixWorld)` / velocity via world quaternion in local mode) onto `this.deathScratch: number[]`. After the loop: resolve target once; skip if unresolved, `=== this` is **allowed** (deferred = safe), skip entirely when `this.isPrewarming` flag set by `prewarmSimulation()`; fire `sub.emitBurstAt(pos, subEmitterBurstCount, vel * subEmitterInheritVelocity)` per death; clear scratch.

Chained sub-emitters are allowed and naturally bounded by each system's `maxParticles` pool (spawn loop only claims inactive slots — lines 314–322 pattern).

### D4 — Camera plumbing: new `syncRenderState()`, keep `applyBillboard` as thin delegate

Trails need camera **position**; `applyBillboard` only receives a quaternion. Adding a param to `applyBillboard` would ripple into `Sprite3D`/`AnimatedSprite3D` call sites.

**Chosen:** new public method on `Particles3D`:
```ts
syncRenderState(cameraQuaternion: Quaternion, cameraPosition: Vector3): void {
  this.syncWorldCompensation();          // late-latch D1 (world mode only)
  this.applyBillboardInternal(cameraQuaternion);  // existing 236–262 body
  this.updateTrailGeometry(cameraPosition);       // D2, no-op unless trailEnabled
}
applyBillboard(q: Quaternion): void { this.syncRenderState(q, ZERO_FALLBACK) }  // deprecated, keeps DeepCore user scripts compiling
```
Both hosts switch their `Particles3D` branch to `syncRenderState(quat, camera.getWorldPosition(scratch))`; `Sprite3D`/`AnimatedSprite3D` branches untouched.

## 5. Open risks

1. **`simulationSpace: 'world'` behavior flip** — any externally-saved scene with the field authored to `world` changes from (buggy) local-follow to true world. No migration path chosen; must be in PR notes + docs. In-repo scan found zero affected scenes.
2. **Trail CPU cost** — full ribbon rebuild each rendered frame is O(active × segments); fine at defaults (512×16), degrades at 10 k×64. Mitigation is documentation + the 64-segment clamp; a follow-up could skip rebuild when `activeCount === 0`.
3. **One-frame world-compensation lag** — if an ancestor moves after `syncRenderState` (e.g. physics writes in a later phase), particles jiggle 1 frame. The double latch (tick + render sync) covers script/animation order; a post-physics mover would need a third hook (defer).
4. **Sub-emitter resolution in play-mode clones** — SceneRunner's serialize→parse clone regenerates the graph but preserves node ids, and `resolveTarget`'s root-walk fallback works pre-`scene`-injection (same guarantee VirtualCamera3D relies on). If ids are ever remapped by prefab instancing (`effectiveLocalId` machinery in SceneSaver 643–666), a prefab-internal sub-emitter reference could break — test with a prefab before shipping if prefabs + particles are a real combination.
5. **Trail material blending** — hardcoded additive (matches the particle material at 145–152); dark trails on bright backgrounds are impossible until a `blending` knob ships (out of scope; note in docs).
6. **`preview` ergonomics** — in-editor, a sub-emitter target only simulates if its own `preview` is on; consider a follow-up where `tickParticlePreview` also ticks nodes referenced as sub-emitters.

# Design spec: Built-in animatable shader effects on GeometryMesh (dissolve, rim light, UV-scroll, flash-tint)

Engine-level change (runtime + serialization + docs) — per CLAUDE.md this plan **requires user confirmation before implementation**, then `cd packages/pix3-runtime && npm run yalc:publish` + `yalc update` in DeepCore, and a `docs/node-types-reference.md` update.

## 1. Decisions (each fork, with grounding)

**D1 — Target GeometryMesh only.** Confirmed it is the sole node authoring an editable `MeshStandardMaterial` (`packages/pix3-runtime/src/nodes/3D/GeometryMesh.ts:72`, `_stdMaterial` getter at :318). `MeshInstance` uses imported GLTF materials, `InstancedMesh3D`/`Sprite3D` are different material shapes. No abstraction layer; if effects later move to MeshInstance, extract the injector then.

**D2 — Injection via `onBeforeCompile` + `material.defines` + `customProgramCacheKey`.** Precedent: `Particles3D.configureMaterialForInstanceAlpha` (`packages/pix3-runtime/src/nodes/3D/Particles3D.ts:460-485`). Improvement over Particles3D: gate each effect's GLSL with `#ifdef PIX3_FX_*` driven by `material.defines`. Verified in `node_modules/three/src/renderers/webgl/WebGLPrograms.js`: `material.defines` is read for **all** materials (line 187) and folded into the program cache key (lines 398-403), so toggling an effect recompiles correctly and disabled effects cost zero GPU. `customProgramCacheKey = () => 'pix3-gmesh-fx-v1'` exists only to version the injected GLSL text (the cache key does not hash the onBeforeCompile closure); bump on any GLSL change. Param changes are uniform-only — no recompile.

**D3 — FLAT schema props, not a nested object sub-schema (roadmap wording is wrong — call this out to the user).** Two hard confirmations:
- The animation timeline only offers track types in `SUPPORTED_TRACK_TYPES = {number, boolean, string, vector2, vector3, euler, color}` — `'object'` is absent, so an object-typed prop is **silently non-animatable** (`src/ui/animation-timeline/animation-timeline-panel.ts:94-103`).
- The runtime clip binder resolves only top-level schema names: `getPropertyDefinition(getNodePropertySchema(node), track.property)` (`packages/pix3-runtime/src/animation/clip-evaluator.ts:335`); there is no dotted-path support in `PropertyTrack.property` (`animation/keyframe-types.ts:47-48`).
So every effect param is a flat `fx*`-prefixed `number`/`color`/`vector2`/`boolean` prop under a new `Effects` UI group. Keyframing then works with zero animation-system changes.

**D4 — Private varying `vPix3Uv`.** `MeshStandardMaterial` only declares `vMapUv` under `USE_MAP`, but the raw `attribute vec2 uv;` is declared **unconditionally** in the WebGLProgram vertex prefix (verified `node_modules/three/src/renderers/webgl/WebGLProgram.js:624`), and onBeforeCompile runs before the prefix is prepended, so prepending `varying vec2 vPix3Uv;` to both shader strings and assigning after `#include <uv_vertex>` is safe with or without any map bound.

**D5 — Play-mode isolation is free.** `SceneRunner` clones via `sceneManager.serializeScene(sourceGraph)` → `parseScene(serialized)` (`packages/pix3-runtime/src/core/SceneRunner.ts:172-173`); the loader's GeometryMesh branch (`SceneLoader.ts:1481-1533`) constructs a fresh node, whose constructor builds a fresh `MeshStandardMaterial` (`GeometryMesh.ts:72`). Effect state serialized in `serializeConfig()` (read from live node state, same rule as the color fix documented at `SceneSaver.ts:528-533`) is re-applied in the clone's constructor. Nothing shared; stopping play discards the clone.

**D6 — Time source: `tick(dt)` override, play-mode only.** `NodeBase.tick` recurses `NodeBase` children (`nodes/NodeBase.ts:308-330`; the raw three `Mesh` child is skipped by the `instanceof NodeBase` guard) and SceneRunner ticks roots each frame (`SceneRunner.ts:379`). UV-scroll accumulates a `uPix3UvOffset` vec2 CPU-side in `tick` (offset += speed·dt, wrapped mod 1) — speed changes mid-play don't jump, and there's no `uTime` to desync. Consequence: UV-scroll is static in the edit-mode viewport (no ticks outside play; the ≤500 ms heartbeat wouldn't advance it anyway). Accepted; all other effects preview live in the editor because inspector/timeline-preview edits go through schema setters → uniforms (existing preview machinery already `requestRender`s).

**D7 — UV-scroll needs an albedo map to be visible.** GeometryMesh currently has **no** `map` — only color/roughness/metalness/aoMap (`GeometryMesh.ts:67-75`). ## 1. Decisions (each fork, resolved)

| Fork | Decision | Why |
|---|---|---|
| Bus topology | 3 fixed buses `'master' \| 'music' \| 'sfx'`; each = `GainNode → BiquadFilterNode(lowpass)`; music/sfx feed master's input; master's filter feeds `destination`. No user-defined buses in M. | Fixed union type keeps schema enums, YAML, and API trivially typed; Godot-style bus editor is a later milestone. |
| Filter always in graph | Yes — a *transparent* lowpass (`frequency = 20000`, `Q = 0.7071` Butterworth, no resonance bump) permanently wired per bus. | Snapshots become pure `AudioParam` ramps; no graph rewiring (which clicks) at snapshot time. A 20 kHz Butterworth lowpass is inaudible. |
| Randomization location | Inside `AudioService.play()` via new `pitchVariation` / `volumeVariation` options. | Four call sites (PlaySoundBehavior, AudioPlayer, AnimationPlayerBehavior audio keys, `scene.audio.play`) get it for free; diagnostics snapshot records the *effective* randomized values. |
| Variation semantics | Linear fraction: `rate *= 1 + rand(-v,+v)` clamped `≥0.01`; `volume *= 1 + rand(-v,+v)` clamped `≥0`. UI range 0–1. | Matches the existing linear `playbackRate` plumbing; semitone math adds no value at this tier. |
| Snapshot model | Named `AudioSnapshot { name; lowpassHz?: Partial<Record<Bus, number>>; volumeScale?: Partial<Record<Bus, number>> }`, registry inside AudioService with built-ins `'default'` and `'muffled'`. Effective bus gain = `userBusVolume × snapshotVolumeScale`. | User volume (mixer) and snapshot (state) must compose, not overwrite — otherwise leaving a snapshot forgets the user's mix. |
| `'muffled'` targets | `{ lowpassHz: { master: 700 }, volumeScale: { master: 0.85 } }` — master only. | Slow-mo should muffle *everything*; per-bus filters remain for future snapshots (e.g. "pause menu" muffling sfx but not music). |
| Muffle policy owner | **SceneRunner**, not AudioService: one guarded comparison per tick against `gameTime.baseScale`. | AudioService stays engine-generic (also used by editor previews); SceneRunner already owns the GameTime coupling. |
| Slow-mo signal | New `GameTime.baseScale` getter (slow-mo blend value, hitstop-independent). **Never** `scale` — hitstop forces `scale` to 0, so every 80 ms micro-hitstop would pump the filter. | Requirement (5); the field already exists privately. |
| Default bus | `'sfx'` for all playback when unspecified. | Everything routes through master anyway; music is opt-in via the new `bus` property on AudioPlayer. |
| Facade | `SceneService.get audio(): AudioApi` — lazy cached, mirroring `get juice()` (SceneService.ts L151–156). Null-safe when no delegate. | Requirement (4); consistent with `scene.time` / `scene.juice`. |

```ts
private readonly buses = new Map<AudioBusName, AudioBus>();
private readonly snapshots = new Map<string, AudioSnapshot>();
private activeSnapshotName = 'default';
private static readonly LOWPASS_OPEN_HZ = 20000;

// constructor, replacing L66-67:
const master = this.createBus(this.context.destination);
const music  = this.createBus(master.input);
const sfx    = this.createBus(master.input);
this.buses.set('master', master).set('music', music).set('sfx', sfx);
this.snapshots.set('default', { name: 'default' });
this.snapshots.set('muffled', { name: 'muffled',
  lowpassHz: { master: 700 }, volumeScale: { master: 0.85 } });

private createBus(target: AudioNode): AudioBus {
  const input = this.context!.createGain();
  let filter: BiquadFilterNode | null = null;
  if (typeof this.context!.createBiquadFilter === 'function') {
    filter = this.context!.createBiquadFilter();
    filter.type = 'lowpass';
    filter.frequency.value = AudioService.LOWPASS_OPEN_HZ;
    filter.Q.value = 0.7071;
    input.connect(filter); filter.connect(target);
  } else {
    input.connect(target);            // degraded but functional (mock/legacy)
  }
  return { input, filter, userVolume: 1 };
}

setBusVolume(bus, volume, fadeSec = 0.05): void {
  const entry = this.buses.get(bus); if (!entry || !this.context) return;
  entry.userVolume = Math.max(0, Number.isFinite(volume) ? volume : 1);
  this.applyBusGain(bus, Math.max(0.001, fadeSec / 3));   // τ ≈ fade/3 → ~settled at fadeSec
}
private applyBusGain(bus, tc): void {
  const entry = this.buses.get(bus)!;
  const scale = this.snapshots.get(this.activeSnapshotName)?.volumeScale?.[bus] ?? 1;
  entry.input.gain.setTargetAtTime(entry.userVolume * scale, this.context!.currentTime, tc);
}

applySnapshot(name, options): void {
  const snap = this.snapshots.get(name);
  if (!snap || !this.context) { if (!snap) console.warn(`[AudioService] Unknown snapshot "${name}".`); return; }
  this.activeSnapshotName = name;
  const tc = Math.max(0.001, options?.timeConstantSec ?? 0.08);
  for (const busName of AUDIO_BUS_NAMES) {
    this.applyBusGain(busName, tc);
    const filter = this.buses.get(busName)!.filter;
    filter?.frequency.setTargetAtTime(
      snap.lowpassHz?.[busName] ?? AudioService.LOWPASS_OPEN_HZ, this.context.currentTime, tc);
  }
}

// play() changes:
// after L170:  playbackRate *= randomized; volume *= randomized (clamps as §1)
const pv = clamp01(options?.pitchVariation ?? 0);
const vv = clamp01(options?.volumeVariation ?? 0);

const volume = Math.max(0, (options?.volume ?? 1) * (1 + (Math.random()*2-1) * vv));
```

## 11. Open risks

1. **Existing-scene compatibility**: old YAML has no `bus` — loader defaults to `'sfx'`, which changes nothing audibly (sfx → master at gain 1, transparent filter). Safe.
2. **Editor previews share the AudioService singleton** (`AnimationTimelinePreviewService`, `GamePlaySessionService`): a crash between `applySnapshot('muffled')` and `stop()` would leave editor audio muffled until the next play/stop. `resetBuses()` in `stop()` covers the normal path; acceptable residual risk.

5. **AudioPlayer latent persistence bug fix changes behavior**: scenes whose YAML and live fields disagreed (edited but "never saved") will now save the live values — that is the correct behavior, but flag it in the changelog.
6. **BiquadFilter on 3 buses adds ~nothing** perf-wise (3 IIR filters), but keep an eye on Safari's `createBiquadFilter` availability — it's universal, the feature-detect is only for the test mock and exotic embeds.

# Design Spec — Cutscene Director API (`scene.cutscene.playCinematic`)

All paths under `c:/Projects/pix3-stuff/pix3/`. Grounded in: `packages/pix3-runtime/src/core/SceneService.ts` (flash/fade overlay pattern, delegate at L44-59, `juice` lazy getter at L151), `core/SceneRunner.ts` (delegate binding L788-834, `stop()` L235-297, owns `inputService` L64/114), `core/InputService.ts`, `animation/AnimationPlayerBehavior.ts`, `behaviors/CameraBrainBehavior.ts`, `core/JuiceApi.ts` + `JuiceApi.spec.ts` harness.

---

## 1. Decisions on each fork

**D1 — Where the director lives: `packages/pix3-runtime/src/core/CutsceneApi.ts`, exposed as a lazy `SceneService.cutscene` getter.** Exactly mirrors `JuiceApi` (`SceneService.ts` L151-156): constructed once with the `SceneService` instance, reaches everything through it. No node type, no serialization, no editor UI — this is a pure scripts-facing runtime API (like `scene.time` / `scene.juice`), so the "engine node" confirm-first rule and persistence wiring don't apply.

**D2 — What `id` means: a node query (id / name / slash-path, via `SceneService.findNode`) naming the node that hosts the cinematic's `core:AnimationPlayer`.** Resolution: `findNode(id)` → `node.getComponent(AnimationPlayerBehavior)` (NodeBase L271), falling back to a DFS over the node's descendants for the first `AnimationPlayerBehavior`. The clip to play is `options.clip` (defaults to the player's `play()` default: autoplay clip or first clip — AnimationPlayerBehavior L176-192). Camera moves, VFX, and gameplay beats are authored as keyframe tracks + event tracks on that clip (event tracks are already "the cutscene glue", spec §724).

**D3 — Input locking: depth-counted `lock()`/`unlock()` on `InputService`, with the DOM handlers early-returning while locked and `lock()` clearing transient state.** Chosen over read-masking because it is leak-proof and needs no public-field→getter conversions (`isPointerDown`, `activePointerId` stay plain fields; `@pix3/runtime` is consumed by DeepCore, keep the surface stable). While locked: `onPointerDown/Move/Up`, `onKeyDown/Up` return immediately (so nothing enters `pendingPointerEvents`/`pendingKeyEvents`/`buttons`); `onWheel` and `onContextMenu` still `preventDefault()` (page must not scroll behind a cutscene) but accumulate nothing. `lock()` at depth 0→1 force-releases transient state: `isPointerDown=false`, release+null `activePointerId`, `setButton('Action_Primary', false)`, clear both pending queues and `wheelDelta`. This means UI controls (which poll `input.isPointerDown` / `pointerPosition` in `Node2D.getPointerWorldPosition`, L217-260) and gameplay scripts (`getAxis`/`getButton`/`pointerEvents`) all go quiet with zero per-consumer changes; ECS systems are covered too since `ECSService.beginScene(sceneService, inputService)` (SceneRunner L222) holds the same instance. Held keys can't get stuck: `lock()` cleared them, and keys still held after `unlock()` re-assert via OS key-repeat `keydown`s. `detach()` resets `lockDepth = 0` so a stopped scene never leaks a lock into the next `startScene` (the `InputService` instance is reused across runs).

**D4 — Skip gesture bypasses the lock via raw DOM listeners owned by CutsceneApi.** `canvas.addEventListener('pointerdown', onSkipPointer)` + `window.addEventListener('keydown', onSkipKey)` (defaults: `Escape`, `Space`, `Enter`). These are independent listeners, so the InputService lock is irrelevant to them. **Load-bearing ordering detail:** register the pointer listener in the bubble phase (no `capture: true`). InputService attached its canvas listeners first (`startScene` L203), so on the skipping tap InputService's `onPointerDown` runs *first*, sees `locked`, and drops the event — *then* the skip handler runs. If the skip handler ran in capture phase and synchronously unlocked, the same tap would leak into gameplay as `Action_Primary`. (With D3's early-return + our unlock happening after InputService's handler already returned, the tap is fully swallowed either way — but keep bubble phase and a code comment.) Arming: `skippableAfter` is compared against `performance.now()` captured at play start — real time, no timer object needed. `skippableAfter: undefined` ⇒ never skippable; `0` ⇒ immediately.

**D5 — Looping clips never emit `animation_finished` → guard the completion promise.** `AnimationPlayerBehavior.onUpdate` only emits `animation_finished` on the non-loop branch (L151-156). At `playCinematic` time, resolve the actual `KeyframeClip` via `findKeyframeClip(player.getAnimationSet(), clipName)` and check `clip.loop`: if looping, `console.warn('[Cutscene] clip "…" loops — cutscene will only end via skip()/stop()')` and do **not** wire the signal-based completion; the promise settles only through skip/stop. For non-looping clips, `hostNode.connect('animation_finished', this, handler)` and inside the handler verify the emitted `clipName` matches ours (another clip started on the same player must not complete the cutscene). Always `disconnect` in teardown.

**D6 — Letterbox + all director timing run on REAL time (`performance.now()` + `requestAnimationFrame`), exactly like `flash()` (SceneService L452-462).** The cutscene *content* (the clip) correctly runs on scaled `dt` through `node.tick`; the *chrome* (bars sliding in/out, skip arming) must not freeze under `scene.time.hitstop()` or stretch under slow-mo. Bars are two `div`s appended to `canvas.parentElement` (same mount/positioning logic as `ensureFadeOverlay`, including the `position: relative` fixup), `position:absolute; left:0; right:0; background:#000; pointer-events:none; z-index:9998` — top bar `top:0`, bottom bar `bottom:0`, **below** the fade overlay (9999) and flash (10000) so `fadeToBlack` still covers everything. Heights animate `0% → letterboxSize*100%` via rAF; using CSS percentages makes viewport resizes free.

**D7 — `blendDuration` = one-shot CameraBrain blend override at entry and at exit.** New method on `CameraBrainBehavior`: `overrideNextBlend(durationSec, easing?)` sets a pending override consumed by the *next* `beginBlend` (which fires whenever the active vcam changes, L94-97). Director arms it right before `player.play()` (catches the cut *into* the cinematic vcam whenever the clip raises its priority) and again right before finishing/skipping (catches the cut back to the gameplay vcam when the end pose restores priorities). The override wins even when `config.blendEnabled === false` — an explicit API call beats the authored default. No brain / no vcams (pure-2D scene) ⇒ best-effort no-op. Brain lookup: `scene.getActiveCamera()?.getComponent(CameraBrainBehavior)`.

**D8 — Skip must not drop authored gameplay beats.** `AnimationPlayerBehavior.seek()` only applies value tracks (`applyClipAtTime`, L211-221) — event/audio keys between the current time and the end would be silently lost, and cutscene clips legitimately carry state-changing event keys ("spawn boss", "unlock door"). Add `AnimationPlayerBehavior.finish(fireRemainingKeys = true)`: fire the `fireTimeWindow(binding, time, clip.duration, pendingStartAudio)` window, apply the end pose, stop, emit `animation_finished`. For looping clips `finish()` degrades to `stop()` + emit (there is no "end"). `playCinematic` exposes `fireSkippedEvents` (default `true`) for the rare purely-visual cutscene.

**D9 — Teardown on `stop()`/`dispose()` mid-cutscene.** `SceneRunner.stop()` currently only nulls the delegate (L272); the director's rAF handles, DOM bars, window keydown listener and input lock would all leak. Wire: `SceneRunner.stop()` calls `this.sceneService.cancelActiveCutscene()` **early** (before `detachScripts`/graph dispose, so `player.stop()` still targets a live graph), and `SceneService.dispose()` calls `this.cutsceneApi?.dispose()`. `cancelActiveCutscene()` is a null-safe SceneService method that does *not* lazily instantiate the API (`this.cutsceneApi?.stopAll('stopped')`). Hard-stop path: cancel rAFs, remove bar elements immediately (no exit animation), remove skip listeners, `disconnect` the finish signal, `input.unlock()` (only if this cutscene holds the lock), resolve the promise with `'stopped'`.

**D10 — Concurrency: starting a new cutscene while one is active hard-stops the current one first** (resolves `'stopped'`), then starts the new one. One active cutscene at a time; `stopAll` iterates the (single) active handle.

## 2. API surface

```ts
// packages/pix3-runtime/src/core/CutsceneApi.ts  (new file)
import type { KeyframeEasing } from '../animation/easing';

export interface PlayCinematicOptions {
  clip?: string;              // clip name on the resolved AnimationPlayer (default: play() default)
  skippableAfter?: number;    // seconds of REAL time before skip gesture arms; omit = unskippable
  blendDuration?: number;     // seconds; one-shot CameraBrain blend override at entry + exit
  blendEasing?: KeyframeEasing; // optional easing for the override (default: target vcam's own)
  letterbox?: boolean;        // default true
  letterboxSize?: number;     // fraction of viewport height per bar, clamped 0..0.45, default 0.1
  letterboxEnterSec?: number; // bar slide in/out duration, REAL time, default 0.4
  lockInput?: boolean;        // default true
  fireSkippedEvents?: boolean;// default true — on skip, fire remaining event/audio keys (D8)
  skipKeys?: string[];        // KeyboardEvent.code list, default ['Escape','Space','Enter']
}

export type CutsceneOutcome = 'finished' | 'skipped' | 'stopped';

export interface CutsceneHandle {
  readonly done: Promise<CutsceneOutcome>; // never rejects; resolves 'stopped' on failure-to-start
  readonly isActive: boolean;
  skip(): void;   // programmatic skip — ignores skippableAfter gating
  stop(): void;   // hard abort: player.stop() (pose kept), instant bar removal, unlock
}

export class CutsceneApi {
  constructor(private readonly scene: SceneService) {}
  playCinematic(id: string, options: PlayCinematicOptions = {}): CutsceneHandle;
  stopAll(outcome: CutsceneOutcome = 'stopped'): void; // called by SceneRunner.stop()
  dispose(): void;                                     // stopAll + drop DOM refs
}
```

### `playCinematic` control flow (pseudocode)

```
playCinematic(id, o):
  active?.stop()                                  // D10
  node = scene.findNode(id);  player = node ? findPlayerOn(node) : null
  if (!player || !node) { warn; return inertHandle('stopped') }   // resolved promise
  clipName = o.clip ?? player default; clip = findKeyframeClip(set, clipName)
  if (!clip) { warn; return inertHandle('stopped') }

  input = scene.getInputService()
  if (o.lockInput !== false && input) { input.lock(); holdsLock = true }
  if (o.letterbox !== false) ensureBars(); animateBars(0 -> size, enterSec)   // rAF + performance.now
  startedAt = performance.now()
  attachSkipListeners()                            // canvas pointerdown (bubble) + window keydown
  brain()?.overrideNextBlend(o.blendDuration, o.blendEasing)   // entry blend (if provided)

  if (!clip.loop) hostNode.connect('animation_finished', this, onFinished)  // clipName-checked
  else warn('loops — ends only via skip()/stop()')
  player.play(clipName) || teardown('stopped')

onSkipGesture:            // pointer: canvas only; key: e.code in skipKeys
  if (o.skippableAfter == null) return
  if (performance.now() - startedAt < o.skippableAfter * 1000) return
  skip()

skip():
  brain()?.overrideNextBlend(o.blendDuration, o.blendEasing)   // exit blend
  player.finish(o.fireSkippedEvents !== false)                 // fires window + end pose + signal
  // onFinished fires from finish()'s emit — but settle idempotently with outcome 'skipped'
  settle('skipped')

onFinished(clipName): if (clipName === ours) settle('finished')

settle(outcome):        // idempotent — first caller wins
  detachSkipListeners(); hostNode.disconnect('animation_finished', ...)
  if (holdsLock) input.unlock()
  animateBars(current -> 0, enterSec, then removeBars())       // async retract; control is back NOW
  resolve(outcome)

stop() / stopAll():     // hard path (SceneRunner.stop, dispose, D10 preemption)
  player?.stop()        // pose kept, no signal
  cancel bar rAF; removeBars() immediately; detach listeners; disconnect signal
  if (holdsLock) input.unlock()
  resolve('stopped')
```

Note the `skip()` ordering: `overrideNextBlend` is armed **before** `player.finish()`, because `finish()` applies the end pose (typically dropping the cinematic vcam's priority), which makes `CameraBrainBehavior.pickActive` flip on the next `onUpdate` and consume the override in `beginBlend`.

## 5. Open risks

1. **Cutscene authoring contract**: return-of-control assumes the clip's final keyframes restore gameplay vcam priorities (or the cinematic vcam's `visible`). The director can't know the "gameplay camera" — it only smooths the cut (D7). Document in node-types-reference.
2. **Direct DOM listeners in user scripts bypass the lock** by construction (same loophole the skip gesture exploits). Documented limitation; the whole polled-input surface is covered.
3. **Hover during lock**: `UIControl2D.updateInteractionState` still tracks hover from the frozen `pointerPosition` (position stops updating while locked, so hover is stale but inert — no clicks possible). Cosmetic; accepted.
4. **Clip runs on scaled dt**: a `hitstop()` fired by an event key inside the cutscene freezes the clip but not the skip arming/letterbox (by design, D6). If a game sets `time.setScale(0)` and never restores it, an unskippable cutscene stalls forever — same hazard as any `scale=0` misuse; `skippableAfter` is the mitigation.
5. **`SceneServiceDelegate.getInputService()` is a breaking interface change** for any out-of-repo delegate implementor. Checked: only `SceneRunner` + the two runtime spec files implement it (DeepCore's `DeepCoreRunner.ts` only *calls* `SceneService` methods via a structural type). Safe, but flag in the yalc-publish notes.
6. **`animation_finished` identity check** relies on clip-name equality; two clips with the same name on the same player would alias — pre-existing `KeyframeAnimationSet` invariant (names are unique per set), not new risk.
7. **Skip pointerdown ordering** (D4) depends on InputService attaching its canvas listeners before `playCinematic` runs — always true (`startScene` L203 precedes any script `onStart`). Guard with a comment; the D3 early-return makes it defense-in-depth rather than load-bearing.

# Design Spec — In-Engine Gameplay Video Recording (real-time path, roadmap M)

## 0. Scope decision

Real-time only: `MediaRecorder` over `canvas.captureStream(fps)` + Web Audio tap, saved as WebM into the project. The deterministic offline fixed-step path is **explicitly deferred** (needs a new public `SceneRunner` fixed-step API; offline audio is unsolvable with `MediaRecorder`) — nothing below should be built to anticipate it.

## 1. Resolved forks

**Fork 1 — exposing the private canvas/runner from `GamePlaySessionService`.**
Decision: do **not** add getters. Copy the existing `ProfilerSessionService` integration verbatim (`c:\Projects\pix3-stuff\pix3\src\services\GamePlaySessionService.ts` lines 227, 248, 275): inject the new `GameRecordingService` and push `runner`/`renderer` into it at the same seams. This is the established pattern for "editor service needs the live runtime", is popout/tab agnostic, and gives the recording service a deterministic teardown hook (getters would leave the stop/restart race unsolved).

**Fork 2 — audio tap.** `AudioService.createRecordingDestination()` in the runtime package (`packages/pix3-runtime/src/core/AudioService.ts`). Taps `masterGain` (post-master-volume, line 66-67: `masterGain -> context.destination` stays untouched; we add a second fan-out edge `masterGain -> MediaStreamAudioDestinationNode`). Composes with future audio-buses work as long as `masterGain` remains the final node — the tap never re-parents anything. Runtime change → needs `yalc:publish` for DeepCore.

**Fork 3 — mutation gateway.** `game.record` toggle command + `SetRecordingStateOperation`. The operation returns `didMutate: true` **without** a `commit` (legal per `OperationInvokeResult`, `src/core/Operation.ts:28-31`). Deliberate deviation from `SetPlayModeOperation` (which does commit): undoing "start recording" cannot un-start a `MediaRecorder`; recording state is transient hardware state, not document state.

**Fork 4 — blur/focus stalls.** Two independent suppressions (both verified as real stall sources):
- `GamePlaySessionService.handleFocusPause()` (line 322-340) pauses the runner on blur when `pauseRenderingOnUnfocus` — a paused runner stops painting, so `captureStream` emits no frames. Suppress by adding `&& appState.ui.gameRecording.status !== 'recording'` to the `shouldPause` expression. Re-evaluation is free: `initialize()` already subscribes to `appState.ui` (line 92) and `queueSync -> syncRuntimeToUiState -> handleFocusPause`.
- `AudioService.handleActivityChange` (line 102-119) suspends the `AudioContext` on blur — a suspended context makes the recording track silent. Suppress via a tap ref-count inside `AudioService` (see §3).
- **Documented limitation**: this covers *unfocused-but-visible* windows. A fully hidden tab still stalls because Chrome throttles rAF itself; nothing app-level can fix that.

**Fork 5 — resolution presets.** `captureStream` is locked to the canvas backing-store size, and resizing the live game canvas mid-play is invasive. Decision: `native` preset = direct `renderer.domElement.captureStream(fps)` (zero-copy); `1080p`/`720p` presets = a hidden 2D "scaler" canvas, pumped by `runner.subscribeFrameStats(...)` (`SceneRunner.ts:368` — the listener fires synchronously right after `render()` inside `tick()`, so `drawImage` from the WebGL canvas is safe without `preserveDrawingBuffer`), aspect-fit with letterbox, then `scalerCanvas.captureStream(fps)`.

**Fork 6 — saving.** Add `FileSystemAPIService.writeBlobFile(path, blob)` (clone of `writeBinaryFile`, line 477 — `FileSystemWritableFileStream.write` accepts `Blob` natively, avoids doubling memory via `arrayBuffer()`). Save to `res://recordings/<sceneName>-<yyyyMMdd-HHmmss>.webm`, `createDirectory('recordings')` first.

```ts
export interface AudioRecordingTap {
  readonly stream: MediaStream;
  dispose(): void;
}

// fields
private readonly recordingTaps = new Set<MediaStreamAudioDestinationNode>();

createRecordingDestination(): AudioRecordingTap | null {
  if (!this.context || !this.masterGain) return null;         // video-only recording fallback
  const dest = this.context.createMediaStreamDestination();
  this.masterGain.connect(dest);                               // fan-out; destination edge untouched
  this.recordingTaps.add(dest);
  if (this.suspendedByFocusLoss && this.context.state === 'suspended') {
    this.suspendedByFocusLoss = false;
    void this.context.resume();                                // don't start a recording muted
  }
  return {
    stream: dest.stream,
    dispose: () => {
      try { this.masterGain?.disconnect(dest); } catch { /* already torn down */ }
      this.recordingTaps.delete(dest);
      this.handleActivityChange();                             // re-apply focus policy
    },
  };
}
```

## 11. Open risks

- **Hidden tab ≠ blurred window**: Chrome throttles rAF in hidden tabs; recording stalls there regardless of suppression. Document; possibly warn in UI when `visibilitychange → hidden` during recording.
- **Stop-flush race on detach**: `handleRuntimeDetaching` is sync while MediaRecorder flush is async; `requestData()` before `stop()` bounds loss to <1 timeslice. The last ~1s frame may be lost if the WebGL context is destroyed first — acceptable.
- **Encoder cost**: VP9 @1080p60 can dent game fps on weak GPUs; the 30 fps / 720p presets are the escape hatch. Chrome may silently fall back to VP8.
- **WebM seekability**: MediaRecorder output lacks duration/cues (long-standing Chrome issue); players show unknown duration. Deferred polish: post-process with `ts-ebml` before save.
- **Runner pause during recording** (e.g. user toggles `pauseRenderingOnUnfocus` off-path): video shows a frozen frame while wall-clock timestamps advance — correct-but-surprising; no action for v1.
- **Audio coverage**: only Web-Audio routed through `AudioService.masterGain` is captured — currently all pix3 audio, and the audio-buses work must keep `masterGain` terminal for the tap to stay complete (stated assumption).
- **`OperationService.invoke` with commit-less results**: verify `HistoryManager` tolerates `didMutate: true` without `commit` (it should — `commit?` is optional in the contract at `src/core/Operation.ts:30`); if any code path assumes commit-on-mutate, fall back to returning `didMutate: false` semantics with direct field patching inside `perform` (still gateway-compliant).
